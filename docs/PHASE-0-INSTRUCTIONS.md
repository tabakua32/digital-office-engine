# Фаза 0 — Інструкція для користувача

## Звідки запускати Claude

**Завжди** запускай Claude Code з кореня проекту:
```bash
cd ~/Downloads/AI_PROJECT/nanoclow
claude
```

Claude прочитає `nanoclow/CLAUDE.md` → `CONTEXT-MAP.md` → знайде git-репо `digital-office-engine/`.
Git-операції, хуки, скіли — все в `digital-office-engine/`.

## Гілки

| Гілка | Призначення |
|-------|-------------|
| `main` | Стабільна, деплойна версія |
| `skill-factory/base` | Фаза Skill Factory (F-0..F-6) |
| `feat/{назва}` | Нова функціональність |
| `fix/{назва}` | Виправлення багів |
| `docs/{назва}` | Документація |

Для Фази 0 працюй на `main`. Коли почнеш Skill Factory — переходь на `skill-factory/base`.

## Ритуал початку сесії

1. Claude автоматично читає CLAUDE.md
2. Перевір MEMORY.md — що було зроблено раніше
3. Перевір `groups/{dept}/memory/learnings.md` — уроки з минулих сесій
4. Знайди відповідний скіл в `.claude/skills/_index.md`
5. Прочитай скіл повністю перед початком роботи

## Доступні команди

```bash
npm test          # Запуск тестів (vitest)
npm run typecheck # Перевірка типів (tsc --noEmit)
npm run dev       # Запуск в режимі розробки
npm run build     # Збірка проекту
npm run format    # Форматування коду (prettier)
```

## Безпека

- **src/** — НІКОЛИ не редагуй. Це upstream NanoClaw
- **.env** — НІКОЛИ не показуй/читай через Claude. Редагуй вручну
- **Коміти** — після кожного підзавдання, не в кінці сесії
- **Тести** — запускай `npm test` + `npm run typecheck` після кожної зміни
- **Force push** — заблоковано. Не намагайся обійти

## Управління контекстом

| Зона | % контексту | Дія |
|------|-------------|-----|
| Зелена | 0-60% | Працюй вільно |
| Жовта | 60-70% | Плануй компакцію |
| Червона | 70%+ | `/compact` або `/clear` |
| Критична | 90%+ | Тільки нова сесія |

Перевіряй: `/context` або дивись статус-бар

## Протокол відновлення

Якщо сесія зламалась:
1. Створи `HANDOFF.md` — що зроблено, що далі, які рішення
2. `/clear`
3. Нова сесія: "Read ./HANDOFF.md and continue"

## Після Фази 0

Коли Фаза 0 завершена:
1. Повний огляд всього проекту
2. Уточнення ТЗ (технічного завдання)
3. Перехід до Skill Factory F-1 (Taxonomy)
